## 1. 为什么不选传统 CMS（WordPress / Joomla）

Copy Deck v4.1 对技术侧提出了 **5 个非常规需求**，传统 CMS 无法原生满足或成本过高：

1. **Proof Point 三层分级强制约束**（v4.1 §6.2）：Verified / In Development / Available on Request 三类内容在不同页面有不同可用范围，需要内容建模级权限控制，不是"显示/隐藏开关"能解决的。
2. **Placeholder Registry 上线门禁**（v4.1 §6.1）：Case Study 每张卡必须 ≥ 3 个量化数字，未填齐不允许发布。需要 CMS 层的"校验规则 + 工作流阻断"。
3. **禁用表述扫描**（v4.1 §6.3）：zero-hallucination / 100% accurate / industry-leading 等词组必须在发布前自动拦截。WordPress 需要装插件做正则扫描，维护成本高。
4. **/services/other-engagements 特殊 SEO 策略**（v4.1 §5.7）：单页 noindex + sitemap 排除 + 内部链接 nofollow + GA 二次流量标记 + HubSpot Lead Scoring 模型隔离。WordPress 混合多个插件实现，出错率高。
5. **法务 / IR 签字工作流**：每条数据 claim 需逐条签字。需要发布审批流 + 审计日志。

此外：
- **WordPress 安全面太大**：港股上市公司官网被挂马属披露事故；WP 插件生态 CVE 高发。
- **模板痕迹重**：2415.HK + AI 转型品牌定位需要定制化视觉；WP / Joomla 主题即使重度定制，最终被投资者识别为"套模板"的概率仍高。
- **国际访问速度**：WP 动态渲染叠加插件，P75 LCP 难压到 2.5s 以内；投资者反向尽调时常用 PageSpeed Insights 作为"公司专业度"的非正式代理指标。

## 3. CMS 选型对比

### 3.1 候选矩阵

| 维度 | Sanity（首选） | Payload CMS（次选） | Strapi | WordPress Headless |
|---|---|---|---|---|
| 部署模式 | SaaS（全球 CDN） | 自托管 / SaaS | 自托管 / SaaS | 自托管 |
| 内容建模 | Schema as code（TS） | Schema as code（TS） | GUI + JSON | GUI + ACF 插件 |
| 自定义校验规则 | ✅ 原生（Zod 风格） | ✅ 原生（TS hooks） | ⚠️ 需插件 | ⚠️ 需插件 + PHP |
| 发布工作流 | ✅ 原生（Studio） | ✅ 原生 | ⚠️ 需定制 | ⚠️ 需插件 |
| 审计日志 | ✅ History | ✅ Versions | ⚠️ 需插件 | ⚠️ 需插件 |
| 本地化（EN/CN） | ✅ 成熟方案 | ✅ Plugin | ✅ i18n 插件 | ⚠️ WPML 商业插件 |
| 合规 / 数据主权 | 数据托管美国 / EU | 可自托管 | 可自托管 | 可自托管 |
| 学习曲线 | 中 | 中 | 中低 | 低 |
| 长期维护成本 | 低 | 中 | 中 | 高（插件升级频繁） |

### 3.2 首选 Sanity 的理由

1. **Schema as Code**：Copy Deck v4.1 的"Proof Point 三层分级"、"Case Study ≥ 3 metrics"等规则可以在 schema 里直接写成 validation rule，无需二次开发。
2. **Portable Text**：富文本以结构化 JSON 存储，前端可针对数据 claim、禁用词做实时校验渲染。
3. **Preview + Publish Workflow**：原生支持 draft / published 分离，法务 IR 签字前不会误发布。
4. **Reference 字段**：Case Study 里的 "Metric 1/2/3" 可以 reference 到统一的 `Claim` collection，一次签字，多处复用，不怕散落在各页面遗漏审计。
5. **CDN 自带**：Sanity 的 asset 走全球 CDN，图片响应速度对投资者首屏体验友好。

### 3.3 选 Payload 的场景（次选预案）

若法务 / 数据合规要求 **CMS 数据必须自托管**（例如香港办公室要求数据留在特定区域，或避免 SaaS 第三方处理上市公司草稿），则切换 Payload：
- 自托管在 AWS Hong Kong 或 Alibaba Cloud International
- 所有功能与 Sanity 等价，但需额外 DevOps 投入（数据库备份、升级、监控）

### 3.4 不选 Strapi / WP Headless

- Strapi 的 validation rule 需要 custom plugin 开发，工作量与 Sanity 持平但生态更弱。
- WP Headless = WP 运维负担 + Headless 复杂度，最糟组合。

## 4. 前端技术栈

| 层 | 选择 | 理由 |
|---|---|---|
| 框架 | **Next.js 14（App Router）** | SSG + ISR 满足 SEO；RSC 降低 JS bundle；Vercel 原生支持 |
| 语言 | TypeScript strict | 与 Sanity schema 类型贯通，降低 claim / proof point 错配风险 |
| 样式 | Tailwind CSS + shadcn/ui | 定制化视觉不套模板；设计 token 与品牌系统对齐 |
| 富文本渲染 | @portabletext/react | 匹配 Sanity Portable Text，支持自定义 serializer（数据 claim 自动加脚注） |
| 动效 | Framer Motion（克制使用） | 避免"AI 官网常见的过度动画"印象 |
| 表单 | React Hook Form + Zod | 与 HubSpot Forms 集成；Smart Form 分支路由 |
| 图表（如 IR 引用） | Recharts（轻量） | 若首页 Trust Bar 要可视化 3.33M+ 数据 |
| 分析 | GA4 + HubSpot + Vercel Analytics | 多源交叉验证流量口径 |

**不使用** Wix / Webflow / Framer：这些工具快速出站，但无法承载 Copy Deck v4.1 的发布工作流、校验规则与合规审计要求。

## 6. 合规与审核工作流（最关键）

Copy Deck v4.1 §6.4 定义了四个 Go-Live Gate，以下是技术实现方案：

### 6.1 禁用表述自动扫描

```
GitHub Action / Pre-publish Hook
├─ 正则规则库：/lint/forbidden-terms.json
│   - zero.?hallucination
│   - 100\%\s+accurate
│   - industry.?leading
│   - best.in.class
│   - world'?s\s+#1
├─ 扫描范围：Sanity draft 内容 + 代码库静态文案
└─ 命中 → PR 阻断 / CMS 禁止 publish
```

### 6.2 Placeholder Registry 发布门禁

Sanity schema 层定义：
```typescript
// Case Study schema 片段
validation: (Rule) => Rule.custom((doc) => {
  const metrics = [doc.metric1, doc.metric2, doc.metric3];
  const unfilled = metrics.filter(m => !m || m.startsWith('⚑'));
  return unfilled.length === 0
    || { message: '3 个量化指标必须全部填齐后才能发布' };
})
```

### 6.3 数据 Claim 审计

新增 `Claim` collection：
- `statement`（文本）
- `source`（URL / 文档引用）
- `year`（年份必填）
- `approvedBy`（法务 / IR 签字人）
- `approvedAt`（时间戳）
- `scope`（可使用范围：`verified` / `inDevelopment` / `onRequest`）

所有页面里引用数据 claim 必须 reference 此 collection，不允许硬编码文本。

### 6.4 Logo / 客户名授权追踪

新增 `ClientReference` collection：
- `companyName`（可能脱敏为"Global Top-10 Medtech"）
- `logoAuthorized`（布尔）
- `authorizationDocument`（附件上传）
- `anonymized`（若未授权则强制勾选）

## 7. 表单与线索路由（Smart Form）

Copy Deck v4.1 多处提到 Smart Form：首页、Entering China §2.10、Going Global §3.10、Other Engagements §5.6。

### 7.1 Smart Form 分支逻辑

```
用户选择意图
├─ Entering China           → Pilot 漏斗 + HubSpot Lead Score A
├─ Going Global (US)        → Pilot 漏斗 + HubSpot Lead Score A
├─ Cross-Border Sprint      → Pilot 漏斗 + HubSpot Lead Score B（低门槛）
└─ Other Engagements        → 销售经理直分配 + HubSpot Lead Score N/A
                              （不进 Pilot 漏斗 KPI）
```

### 7.2 实现

- React Hook Form 控制 UI
- Submit 走 HubSpot Forms API（而非 iframe 嵌入），保证事件可自定义
- 服务端 Next.js Route Handler 做 UTM / 来源页 / 分支标签注入
- Other Engagements 页面的 Smart Form 默认预选分支，GA 标注 `traffic_category=secondary`

## 9. SEO & 可达性

### 9.1 SEO 必做项

- 每页自定义 `<title>` / `<meta description>` / OpenGraph 卡片（Sanity 字段）
- JSON-LD 结构化数据：`Organization`、`Service`、`Article`
- 生成 `sitemap.xml`（**必须排除** `/services/other-engagements`）
- `robots.txt` 控制爬取
- 关键页面 Lighthouse 目标：Performance ≥ 90 / Accessibility ≥ 95 / SEO = 100

### 9.2 /services/other-engagements 特殊处理（v4.1 §5.7）

- 页面 meta：`<meta name="robots" content="noindex, nofollow">`
- 不进 sitemap.xml
- 所有指向该页的内部链接加 `rel="nofollow"`
- GA4 自定义事件标记 `page_category=secondary`
- HubSpot Lead Scoring 规则排除来源 URL 匹配 `/services/other-engagements.*`

### 9.3 无障碍（WCAG 2.1 AA）

国际投资者与美国合作伙伴越发重视 a11y：
- 对比度 ≥ 4.5:1
- 键盘全链路可达
- 所有图片 `alt` 文本
- 表单 label 正确关联

### 11.8 档位推荐与决策建议

**推荐档位 B（标准版，25–40 万 RMB）**

理由：
1. 2415.HK 港股上市公司身份要求官网承担反向尽调任务，档位 A 的"拼装感"风险与上市公司品牌不匹配
2. 档位 C 的 VI 全量升级在当前阶段属于过度投入，应等到下一次重大披露节点或并购窗口再做
3. 档位 B 保留了 Pilot 子页独立（支持广告投放）、合规自动化（港股红线）、CN 反向尽调最低必需（首页 + About）
4. AI 辅助开发能把档位 B 的实际交付质量逼近档位 C 的 80%，但成本只有其 50–60%

**若预算确实受限**：建议走档位 A，但**保留**合规扫描（1–2 万不砍），并在 V2（6 个月后）升级到档位 B。切勿跨过合规这一项。

## 13. 关键决策点（建议团队拍板）

1. **CMS 是否接受 SaaS（Sanity）？** 若否，转 Payload 自托管，工期 +1.5 周，成本 +6–10 万。
2. **多语言范围**：仅首页 + About，还是全站？当前建议前者。
3. **HubSpot 是否已有席位？** 若无，需同步采购并安排 CRM 配置工期。
4. **品牌视觉**：沿用现有 VI，还是借官网重构做一次升级？后者工期 +2 周，成本 +8–15 万。
5. **是否上 Cloudflare Enterprise WAF？** 上市公司官网被挂马属披露事故，建议至少上 Pro。
6. **Go-Live 目标日**：倒推现在是 W0，若 W12 Go-Live 则目标日需法务 / IR 全程配合。
