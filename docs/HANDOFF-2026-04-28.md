## 6. 关键技术约定 / 坑(新会话务必遵守)

### 6.1 git plumbing 工作流(FUSE 锁的解法)

- 控制器 (Claude) 的 `mcp__workspace__bash` 走 sandbox VM,FUSE 层会让 `.git/*.lock` 创建后无法 unlink → 每次 `git add`/`commit` 留死锁
- **应对**: 不调 `git commit`,改用 plumbing — 路径如下:
  ```bash
  cp .git/index /tmp/msh-idxN
  GIT_INDEX_FILE=/tmp/msh-idxN git add <files>
  TREE=$(GIT_INDEX_FILE=/tmp/msh-idxN git write-tree | tail -1)
  PARENT=$(git rev-parse HEAD)
  COMMIT=$(git -c user.email=yog.young@gmail.com -c user.name="Yog Young" \
    commit-tree "$TREE" -p "$PARENT" -m "msg" | tail -1)
  ```
- 提交对象写到 `.git/objects/` 后,**用户在本机跑** `git update-ref refs/heads/master <COMMIT> <PARENT>`(必须先 `find .git -maxdepth 4 -name '*.lock' -delete` 清死锁;**zsh 不支持 `*.lock` glob,要用 `find`**)
- **打 tag**: 直接 `echo <SHA> > .git/refs/tags/<tagname>`(避免 git 命令再起锁)
- 每个 plumbing commit 前用 `git rev-parse HEAD` 确认 sandbox HEAD = 用户上次 update-ref 后的位置(不一致会导致 parent 错链 → tree 缺文件)

### 6.2 Babel-standalone + localhost 工作流

- **file:// 双击不再可用**(Chrome 收紧 file:// XHR,Babel 加载组件被挡)
- 开发用 `cd prototype && python3 -m http.server 8000` → 浏览器 `http://localhost:8000/...`
- **MutationObserver 反馈环禁用**(W6 hotfix 移除了 6 个 solutions HTML 的 lucide MutationObserver,会冻结渲染);新页只用 `window.addEventListener('load', ...) → setTimeout(... lucide.createIcons, 200)`

### 6.3 路径约定

- **Logo `<img src>`**: 一律 root-relative `/assets/logo/medsci-healthcare-logo.svg`(Header.jsx + Footer.jsx + SolutionHeader.jsx + SolutionFooter.jsx 全部对齐)
- **HTML 内 favicon**: 沿用每文件相对路径(`assets/...` 或 `../assets/...`)
- **Top Nav + Mega + Footer 链**: root-absolute `/case-studies/` `/ai-platform.html` `/insights/` `/about.html` `/contact.html` `/solutions/{slug}.html` `/pilots/{slug}.html` `/legal/{slug}.html` `/services/other-engagements.html`
- **`/contact` 必须带 `.html`**(localhost 无 URL 重写;production 上线时如有 rewrite 可去尾缀)
- **IR 外链**: `https://ir.medsci.cn/en/`(旧的 `medscihealthcare.com/ir` 全清)
- **Button.jsx**: 已扩展支持 `href` prop;如传则渲染 `<a>` 否则 `<button>`。"Talk to an Expert" 按钮 4 处都已接 `/contact.html`
